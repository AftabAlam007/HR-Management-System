WorkBridge: HR Management System

An overview of the project's technical architecture for interview preparation.

1. Frontend (Client-Side)

Technology Stack: React, React Router, Tailwind CSS, Axios

Explanation (English)

The frontend is built using React. It operates as a Single-Page Application (SPA), which means the website runs on a single page and content loads dynamically without requiring full page reloads. For styling, Tailwind CSS is used to build modern, responsive designs quickly. React Router handles the page navigation, and Axios is used to make HTTP requests (GET, POST, etc.) to communicate with the backend APIs.

Explanation (Hinglish)

Frontend React use karke banaya gaya hai. Ye ek Single-Page Application (SPA) hai, jiska matlab hai ki poori website ek hi page par chalti hai aur content dynamically load hota hai. Isse user experience bahut fast aur smooth rehta hai.

  • Styling: Design ke liye Tailwind CSS use kiya gaya hai. Ye ek utility-first CSS framework hai jisse hum direct HTML (JSX) mein hi classes dekar design banate hain. Isse custom CSS likhne ki zaroorat kam padti hai aur development fast hota hai.
    Example: <div class="bg-green-500 text-white p-4 rounded-lg">
  • Routing (Page Navigation): Alag-alag pages (jaise /login, /dashboard, /hr/employees) ke beech navigation React Router library se handle hota hai.
  • API Communication: Frontend, backend se data laane aur bhejne ke liye Axios library ka use karta hai. Ye backend ke REST API endpoints par HTTP requests (GET, POST, PUT, DELETE) bhejta hai.
  • State Management: Simple state management ke liye React ke built-in hooks (useState, useEffect) use kiye gaye hain.
  • Component Structure: Code ko reusable components mein toda gaya hai, jaise Navbar, Footer, DashboardLayout, aur alag-alag modals (ApplicantDetailsModal, CreateHrModal).

2. Backend (Server-Side)

Technology Stack: Java, Spring Boot, Spring Security, Spring Data JPA (Hibernate), Maven

Explanation (English)

The backend is developed using the Spring Boot (Java) framework. It functions as a RESTful API server that processes incoming requests from the frontend, executes the core business logic, and interacts with the database. The architecture follows a standard 3-tier pattern: Controller Layer (handles API endpoints), Service Layer (contains business logic), and Repository Layer (interacts with the database using Spring Data JPA). For security, it uses Spring Security with JWT (JSON Web Tokens) to authenticate and authorize users securely.

Explanation (Hinglish)

Backend Spring Boot (Java) framework mein bana hai. Ye ek RESTful API server hai jo frontend se aane wali requests ko handle karta hai, business logic execute karta hai, aur database se interact karta hai.

Architecture (3-Tier)

  • Controller Layer (@RestController): Ye layer API endpoints define karti hai (e.g., /auth/login, /api/hr/employees). Iska kaam frontend se request lena, usko validate karna, aur aage Service layer ko call karna hai.
  • Service Layer (@Service): Yahan saara main business logic likha hota hai. For example, ek naye user ko register karna, uske liye password generate karna, ya leave request ko approve/deny karna. Ye layer Repository layer se interact karti hai.
  • Repository Layer (@Repository): Ye Spring Data JPA interfaces hote hain jo database se interact karte hain. Humein complex SQL queries likhne ki zaroorat nahi padti. JPA (Hibernate) humare liye queries generate kar deta hai.
    Example: userRepository.findByEmail(email)
  • Model/Entity Layer (@Entity): Ye simple Java classes (POJOs) hoti hain jo database tables ko represent karti hain. Har class ek table hoti hai aur uske variables columns hote hain.

Security

Security ke liye Spring Security aur JSON Web Tokens (JWT) ka use kiya gaya hai.

  1. Jab user login karta hai, toh backend ek encrypted JWT generate karke frontend ko bhejta hai.
  2. Frontend is token ko localStorage mein save kar leta hai.
  3. Har agli protected API request ke saath, frontend ye token Authorization: Bearer <token> header mein bhejta hai.
  4. Backend mein ek JWT filter is token ko validate karta hai. Agar token valid hai, toh user ko request access karne ki permission milti hai.

3. Database

Technology: MySQL (or any relational database like PostgreSQL)

Explanation (English)

All critical application data, including user details, employee records, and HR tasks, is securely stored in a relational database, specifically MySQL. The backend communicates with this database using JPA/Hibernate (ORM), which allows us to manipulate database records using Java objects directly without writing complex raw SQL queries.

Explanation (Hinglish)

Saara data (users, employees, jobs, etc.) ek relational database jaise MySQL mein store hota hai.

  • ORM (Object-Relational Mapping): Backend aur database ke beech mein JPA/Hibernate ORM ka kaam karta hai. Isse humein Java objects (Entities) ke through hi database operations (Create, Read, Update, Delete) karne ki facility milti hai. Humein raw SQL likhne se bachata hai.

4. Overall Workflow (Example: User Login)

The complete flow of a user logging into the system is explained below (English & Hinglish):

1
User Action: The user enters their email and password on the React Login page and clicks "Sign In".
(User, React app ke Login page par apna email aur password enter karke "Sign In" button par click karta hai.)
2
Frontend (React): The axios library sends a POST request with the credentials in JSON format to the backend's /auth/login endpoint.
(axios library ek POST request backend ke /auth/login endpoint par bhejti hai.)
3
Backend (Spring Boot): The AuthController receives this HTTP request.
(AuthController is request ko receive karta hai.)
4
Backend (Service & Repo): The AuthService uses UserRepository to verify if the user exists in the database and if the passwords match.
(AuthService, database mein check karta hai ki credentials sahi hain ya nahi.)
5
Backend (Security): If the credentials are correct, JwtService generates a secure JWT (JSON Web Token).
(Agar credentials sahi hain, toh ek naya JWT generate hota hai.)
6
Backend (Response): The Controller sends this JWT back to the frontend as an HTTP response.
(Controller, frontend ko response mein ye JWT bhej deta hai.)
7
Frontend (React): The React app saves this token in localStorage and redirects the user to their dashboard based on their role.
(React app is token ko localStorage mein save karke dashboard par redirect kar deta hai.)
8
Future Requests: For any future requests to protected routes, this JWT is attached to the Authorization header so the backend can identify the user.
(Ab har API call ke Authorization header mein ye JWT bheja jayega.)
